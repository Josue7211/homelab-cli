  <action>
For each of the 14 CLI binaries, perform this audit procedure:

1. **Extract case statement commands**: Find the main case "$1" (or similar dispatch) block. List every command label (e.g., "status", "logs", "restart") excluding catch-all patterns like "*", "--help", "-h", "help", and "" (empty/default).

2. **Extract help text entries**: Find the cmd_help() function. List every command mentioned in the help output (the left-hand side of each "command — description" line).

3. **Compare the two lists**:
   - Commands in case statement but NOT in help text = MISSING from help (must add)
   - Commands in help text but NOT in case statement = STALE in help (must remove)
   - Commands with wrong/outdated descriptions = must update description

4. **Fix any discrepancies found**:
   - Add missing commands to the appropriate section in cmd_help() following the existing section-based format
   - Remove stale commands that no longer exist
   - Fix descriptions that don't match what the command actually does
   - Preserve the existing section organization (e.g., STATUS, MANAGEMENT, etc.)
   - Keep the existing formatting style (indentation, alignment, spacing)

5. **Also check for alias discrepancies**: Some commands have aliases defined in the case statement (e.g., "delete|rm"). Ensure aliases are mentioned in help text where they exist (typically shown as "command|alias" or noted in parentheses).

6. **Run syntax check**: After editing each file, run `bash -n bin/<cli>` to verify no syntax errors were introduced.

Do NOT change any command implementations — only modify cmd_help() function content. If a CLI's help text is already perfect (all commands match), leave it unchanged.

Process the CLIs in this order: portainer, arr, homelab, plex, jellyfin, qbt, adguard, opnsense, sab, gluetun, overseerr, vault, firecrawl, koel.

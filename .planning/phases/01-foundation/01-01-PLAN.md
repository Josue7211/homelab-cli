1. **confirm_action** -- Add after the "Output helpers" section (after the `header()` function, around line 31). This is a user-facing prompt helper, so it fits with the output helpers.

```bash
# ── Confirmation helper ─────────────────────────────────────────
# Usage: confirm_action "Delete stack 'myapp'?"
#   Prompts "Are you sure? (y/N)" and exits 1 if user doesn't type y/Y.
#   Respects HOMELAB_YES=1 env var to skip prompt (for scripting).
confirm_action() {
    local msg="${1:-Proceed?}"
    if [[ "${HOMELAB_YES:-0}" == "1" ]]; then
        return 0
    fi
    echo -en "${YELLOW}${msg} Are you sure? (y/N): ${RESET}"
    read -r answer
    case "$answer" in
        [yY]) return 0 ;;
        *) echo "Aborted."; exit 1 ;;
    esac
}
```

2. **api_patch** -- Add immediately after the existing `api_put()` function (after line 117). Follow the exact same pattern as api_put but use PATCH method.

```bash
api_patch() {
    local url="$1" api_key="${2:-}" data="${3:-}"
    local -a headers=(-H "Content-Type: application/json")
    [[ -n "$api_key" ]] && headers+=(-H "X-Api-Key: $api_key")
    curl -sf -X PATCH "$url" "${headers[@]}" -d "$data" 2>/dev/null
}
```

Do NOT modify any existing functions. Only add these two new functions in the specified locations.

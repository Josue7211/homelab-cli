Add 3 new commands to bin/koel after the existing API commands from plan 01:

1. Add cmd_stats (per D-04, KOEL-04):
   - GET /api/data via koel_api GET "/api/data"
   - Parse response with python3 to extract:
     a) Total songs: len(data.get('currentUser',{}).get('songs',[]) or data.get('songs',[]))
        Note: /api/data may return top-level arrays. Adapt to actual response shape.
        Safest: count items in songs/albums/artists arrays from the response.
     b) Total albums: count of albums array
     c) Total artists: count of artists array
     d) Total duration: sum of all song lengths (seconds), format as "X hours Y minutes"
     e) Total file size: not always available -- if songs have a size field, sum and format with human_size style; otherwise omit
   - Display with header "LIBRARY STATS" and labeled rows:
     Songs:     1,234
     Albums:    123
     Artists:   45
     Duration:  123 hours 45 minutes

2. Add cmd_albums (per D-05, KOEL-05):
   - Optional arg: artist name filter ($1, may be multi-word so artist="$*")
   - GET /api/album via koel_api GET "/api/album"
   - Parse with python3:
     a) Load JSON array of albums
     b) If artist filter provided, filter albums where artist_name contains the filter (case-insensitive)
     c) For each album: extract name (or title), artist_name, year (or created_at year), song_count (or length of songs array)
     d) Sort by artist name then album name
   - Display with header "ALBUMS" and table: Title | Artist | Year | Songs
   - Show count at bottom: "N albums"

3. Add cmd_artists (per D-06, KOEL-05):
   - GET /api/artist via koel_api GET "/api/artist"
   - Parse with python3:
     a) Load JSON array of artists
     b) For each: name, album_count (or len(albums)), song_count (or len(songs))
     c) Sort by name
   - Display with header "ARTISTS" and table: Name | Albums | Songs
   - Show count at bottom: "N artists"

4. Add case statement entries:
   - stats) cmd_stats ;;
   - albums) cmd_albums "$@" ;;
   - artists) cmd_artists ;;
